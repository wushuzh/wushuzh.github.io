Algorithm

leetcode #283 Move Zeroes 要求将给定数组中，值为 0 的元素挪到最后，同时保持其他非零元素的相对位置不变。

直接的解法是第一遍循环，将所有非零元素依次向前排，记录最后位置，第二次循环将后面的元素清零。

def move_zeros(num: List[int]) -> None:
    last_zero_idx = 0
    for i in range(len(nums)):
        if nums[i] != 0:
            nums[last_zero_idx] = nums[i]
            last_zero_idx += 1
    
    for z in range(last_zero_idx, len(nums)):
        nums[z] = 0

Review

Michael Lynch (2017-7-25) Why a Freelance Editor Was a Gamechanger for My Blog 提到他雇了一个编辑，为他的技术博客提出修改意见。之后其阅读量翻了好多倍。

从网上雇佣一个兼职编辑并不太难，但作者发现为自己雇一个编辑，可以让他获得写作风格上的反馈：写作是可以用到各种场合的重要能力，提高的前提当然就是多练，但之后还必须要有反馈，一般陌生读者犯不着给你的挑错别字和语病，他们会直接关掉网页去看别的，而你的家人同事可能也不太方便给你直接了当的真实反馈。

编辑可以为你挑出你的语病，叙述风格上的坏习惯；他们还可以从一个真正读者的角度，让你意识到哪些自己习以为常的实物，实际上对读者来说缺少必要的解释说明，最最重要的是他的编辑提出了这样一个问题：“你想从这边博文中说明什么？不应仅限于陈述你做过的那些事实，写作是一件有创意的事情，要保持有趣。让你的读者开心的笑出来，让他们有欲望读下去”。作者开始像尝试用讲故事方式写文章，结果后续的文章都收到不错的效果。

下面是他对于和找编辑，和他们合作的一些建议：

  • 别挑便宜的，要挑有质量的，为你的编辑的专业程度付费；
  • 要仔细研究过往的兼职的成功率，是否有和你写作内容相关的教育背景或是工作经验；
  • 把东西给编辑前，自己想把文章理通顺，拼写搞对，大声把你的文章读一遍；
  • 别怕批评，别玻璃心；
  • 也不一定全盘接收编辑的所有建议；
  • 开始整理出你的检查清单，以后发文章前逐项核查，形成习惯，不断提高；

Tip

Linux 下为用户设置磁盘限额：

对 ext4：

  • 在 /etc/fstab 中对应分区的记录，要设定相应的的挂载参数：ext4 中 为usrquota;
  • 在对应分区建立数据库 quotacheck -mau ，做相应的开启 quotaon /dev/sdbn;
  • 为用户设定限额：setquota -u usrname softlimit hardlimit softfnum hardfnum
  • 通过编辑器设定: edquota -u user1 按照某个用户的限额设定其他用户：edquota -u user2 -p user1
  • 查看所有用户的使用情况： repquota -uv /dev/sdbn

对 xfs ：

  • 可以设定用户限额uquota，组限额gquota，项目限额pquota，以及不强制选项uqnoenforce；
  • 为用户设定限额 xfs_quota -xc 'limit -u bsoft=100m bhard=200m usr' /partiton11；
  • 进入专家模式，查看所有用户情况 xfs_quota -x: 执行 report -h /partition；
