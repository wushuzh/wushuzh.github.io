Algorithm

leetcode #206 Reverse Linked List 反转链表。

递归方法：

 1def reverse_list(some_head: ListNode) -> ListNode:
 2    if not some_head or not some_head.next:
 3        return some_head
 4
 5    new_head = reverse_list(some_head.next)
 6
 7    some_head_next_node = some_head.next
 8    some_head_next_node.next = some_head
 9    some_head.next = None
10
11    return new_head

首先找到最后一个节点，这次最深的栈调用会在第三行提前返回，new_head 不再更改，依次传回，作为最终结果链表的首节点。

之后依次处理逐层的调用栈：some_head 是之前每次调用栈中逐渐缩短的链表的首元素：将它和它原来指向的下一个元素调换指向关系。

Review

Tip

  • hostname 是老的设定主机名的指令，它更改的是 Transient hostname，若希望持久化，需要配合 /etc/hostname 使用；
  • hostnamectl set-hostname foo是新的设定主机名的指令，能接更改底层文件，甚至可以为机器设定 pretty hostname，保存在 /etc/machine-info 中；

关于主机名解析的顺序设定，可以查看 /etc/nsswitch.conf 中 hosts 的相关记录：

grep hosts /etc/nsswitch
# files: db files nisplus nis dns
hosts:   files dns

这里的 files 即是 /etc/hosts 文件。

注意文件的格式一定为 IP FQDN ALIAS

原因是通常 FQDN 会和证书绑定，如果颠倒了 FQDN 和 ALIAS 的位置，对访问可能会产生影响。
