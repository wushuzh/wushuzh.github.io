Algorithm

leetcode #448 Find All Numbers Disappeared in an Array 长度为 N 的数组，其元素值一定处于 [1, N] 区间内，但有些元素出现了 2 次——导致其他有些在此范围内的值未能出现。

要求找出未能出现的元素列表。

解法思路：如果给定数组其中的元素没有重复，则它们每一个的值都能对应上一个 idx (idx + 1)。

通过轮循这些值，将其对应的 idx 的值做特殊标记，那么重复的值指向的 idx 元素就会被标记多次。而有些 idx 下的值却没有标记，这些没有被标记的值就是要求的解。

标记方法可以选择加 N，那么原来范围是 [1, N] ，变更为 [1 + N, 2N]。

而对这个加过 N 的值取余数，又可以消除标记的影响——余数仍然可以对应同一数组 idx 位置。

def find_disapppeared_nums(nums: List[int]) -> List[int]:
    N = len(nums)
    for i in range(N):
        nums[nums[i % N] - 1] += N
    
    return [i + 1 for i in range(N) if nums[N] <= N]

Tip

autofs 是用来自动挂载文件系统的工具：它可以在你访问特定配置目录时，按照约定设置，自动挂载(mount)各式文件系统：比如光驱、U盘、远程NFS，而在用户一段时间不访问的时候，自动卸载相应文件系统，在 /etc/autofs.conf 中设定了这些时间间隔。

  • 主配置文件 /etc/autofs.master 里面配置了目录(例如 /misc)，和其对应的配置文件(/etc/auto.misc)的映射关系；
  • 在目录配置文件 /etc/autofs.misc 有各个子目录(例如 cd), 和挂载参数和设备；
yum install autofs

cat /etc/autofs.misc |grep cd
cd     -fstype=iso9660,ro,nosuid,nodev   :/dev/cdrom

systemctl restart autofs

实验：在一台服务器上建立 NFS 服务，在另一台安装 aufofs 服务，自动挂载共享目录：

# NFS server setup
yum install nfs-utils

firewall-cmd --permanent --add-service=nfs
firewall-cmd --reload
systemctl enable --now rpcbind nfs-server

mkdir /share
find /usr/share/doc -name "*pdf" -exec cp {} /share \;
echo "/share *(ro)" >> /etc/exports
exportfs -r # re-share
exportfs -s # give summary display of all shared folder
# NFS Client autofs setup
cat /etc/autofs.misc |grep pdf
pdf    -ro,soft,intr  nfsserver:/share

# option soft means report error when nsfserver is not available
# option intr means request can be interrupted in case nfsserver cannot be reached

cd /misc/pdf
ls *pdf
