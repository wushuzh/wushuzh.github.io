Algorithm

leetcode #169 Majority Element 找出那个出现次数超过一半的元素。

思路是使用 dictionary 的 key 对重复元素计数，计数值超过半数就是希望得到的结果。

def majority_element_using_dict(nums: List[int]) -> int:
    dic = {}
    for i in nums:
        if i not in dic:
            dic[i] = 1
        else:
            dic[i] += 1
        
        if dic[i] > len(nums)//2:
            return i

关于 SELinux 的更全面的总结：

有时我们对 linux 服务的自定义配置，可能会因为缺少增加、变更相应的 selinux 的规则，而无法使用。在测试环境下，我们可能直接全局禁止 selinux 了事，但在正规环境下，这确实是最不推荐的做法。

举个例子，比如我们建立一个 samba 服务，使得 windows 用户可以直接访问服务器的设定一个自定义的共享目录，但仅仅设定 777 的目录权限并不够。

yum install 

mkdir -m 1777 /share # add stick bit so that only owner can delete or renanme the files within folder

tail /etc/samba/smb.conf
[share]
path = /share
writeable = yes

testparm # test the smb.conf syntax is correct and dump all svc settings

systemctl enable nmb smb --now

smbpasswd -a root # set smb passwd for user root 
smbclient -L //localhost # list the share folders
# try to connect and list the content
smbclient //locahost/share MYPASSWD
smb: \> pwd
Current directory is \\localhost\share\

smb: \> ls
NT_STATUS_ACCESS_DENIED listing \*

诊断步骤及修复办法：

首先确认系统的 selinux 的运行模式是 Enforce，查看最新的 audit 日志，确认有相关的selinux记录：

getenforce
ausearch -m avc -t recent(10min)/today/yesterday/boot

暂时的解决办法，使用 chcon 对目录变更 selinux 配置，但这些设定会在后续执行 restorecon 或 relabel 后失效

ls -dZ /share/
chcon -t samba_share_t /share
ls -dZ /share/

永久的方法是使用 semanage 工具增加对目录share的相应规则：

yum whatprovides "*/semanage"
yum install policycoreutils-python

semanage fcontext -a -t samba_share_t '/share/(.*)?'
restorecon -Rv /share

另一个办法是改变 selinux 的开关，这样就不用对 samba 共享目录做逐一设定： 

setsebool -P samba_export_all_rw 1
systemctl restart smb

最后预期将整个系统 selinux 设定为 Permissive，我们可仅仅将 samba 服务设置为 Permissive 模式：

yum install setools-console

seinfo --permissive # list all the services in permissive mode

semanage permissive -a smbd_t # add smb service into this list
seinfo --permissive # check again
