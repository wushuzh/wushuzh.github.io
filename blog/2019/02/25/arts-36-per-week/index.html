Algorithm

leetcode #130 Surrounded Regions：要求将图中可以被 X 全部包围的“内部” O 区域做反转，有点类似围棋中“吃子”的操作。

若是使用 Union-Find 来解决，思路是将边缘的 ‘O’ 都链接到一个虚拟的额外节点上，之后就是常规的操作：将二维数组放入 UF 数据结构中，最后可以判断所有未链接到额外节点的 O 都是可以“被吃”的内部区域，做相应反转即可。

需要注意的是：

  • 第 14，17，20 行的判断条件不可级联，都需要独立判断，边缘的节点和 dummy 节点链接后，也需要查看其右，其下是否有其他可联通的节点，为了防止索引溢出，第 17、20 行的条件要增强；
  • 并查集并不保证 union 操作时的挂接顺序，不能写成 if uf.find_root(i * uf.c + j) != uf.dummyIdx, 这本质上是 Union-Find 的 is_connected 操作；
  • 应该对 Union-Find 数据结构做扁平化，以提高性能；
 1def flip_inside_O_by_UF(board: List[List[str]]) -> None:
 2    if not board:
 3        board = []
 4        return
 5
 6    uf = UnionFind(board)
 7    dummyIdx = uf.r * uf.c
 8
 9    for i in range(uf.r):
10        for j in range(uf.c):
11            if board[i][j] == 'O':
12                ijIdx = i * uf.c + j
13                # connect all boundary nodes to the ONE dummy node

14                if i in (0, uf.r - 1) or j in (0, uf.c - 1):
15                    uf.union(ijIdx, dummyIdx)
16                # check its right neighbor is connected ?

17                if j + 1 < uf.c and board[i][j + 1] == 'O':
18                    uf.union(ijIdx, ijIdx + 1)
19                # check its down neighbor is connected ?

20                if i + 1 < uf.r and board[i + 1][j] == 'O':
21                    uf.union(ijIdx, ijIdx + uf.c)
22
23    for i in range(uf.r):
24        for j in range(uf.c):
25            if uf.find_root(i * uf.c + j) != uf.find_root(uf.dummyIdx):
26                board[i][j] = 'X'

Tip

将某进程固定在某个 CPU core 上，防止进程在不同的 CPU core 间来回做切换。

dd if=/dev/zero of=/dev/null &

# enter top utility
#   `f` then add colume P: Last Used Cpu (SMP)
top

# pin dd process to cpu 3
#  taskset -pc [CORE-LIST] [PID]
taskset -pc 3 $(pidof dd)

# bring cpu 3 offline
echo 0 > /sys/bus/cpu/devices/cpu3/online
# double confirm with lscpu
lscpu

# enter top again to confirm dd is using another cpu core
top
