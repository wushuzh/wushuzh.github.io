Algorithm

leetcode #928 Minimize Malware Spread II 此题和 leetcode #924 Minimize Malware Spread 表述基本一致——都是在某些病毒源节点中，找出去除后能最小化网络病毒扩散的一个节点。不同点在于，此题去除时比较彻底，明确要求将 graph 中有关联的链接也一并去除——原来 N * N 矩阵变为 (N - 1) * (N - 1) 矩阵。

Union Find 数据结构中并没有删除连接关系的操作，更不要说是删除一系列的连接关系。所以本题一个重要的思考角度是，将网络节点分为病毒源节点和非病毒源节点。然后在此基础上进行统计分析，从而评估这些连接关系消失后对网络的影响。

  1. 先将所有非病毒源节点放入 union find 中完成连接；
  2. 逐一求得每个病毒源节点对上述网络中产生影响的独立 group 的列表——用每个 group 的 root 节点作为 group_id；
  3. 当这些受影响的 group 中只有一个病毒源时，清除此病毒源可以“拯救”整个group；
  4. 轮询每个病毒源，找出影响最大的返回即可；
 1def min_malware_spread_II_using_unionfind(graph: List[List[int]],
 2                                          initial: List[int]) -> int:
 3    N = len(graph)
 4    clean = set(list(range(N))) - set(initial)
 5
 6    uf = UnionFind(N)
 7
 8    for i in clean:
 9        for j in clean:
10            if graph[i][j]:
11                uf.union(i, j)
12
13    # global view from virus

14    #  one virus can impact which groups: id by root of a group

15    impact_clean_nodes = collections.defaultdict(set)
16    for v in initial:
17        for i in clean:
18            if graph[v][i]:
19                impact_clean_nodes[v].add(uf.find_root(i))
20
21    # global view from clean nodes

22    #  one clean node can be impacted by how many virus

23    impact_clean_counts = collections.Counter()
24    for v in initial:
25        for i in impact_clean_nodes[v]:
26            impact_clean_counts[i] += 1
27
28    max_impact_num = -1
29    ans = min(initial)
30    # loop with each virus to see

31    #   how many groups it can independently impact

32    for v, some_clean_nodes in impact_clean_nodes.items():
33        cnt = 0
34        for i in some_clean_nodes:
35            if impact_clean_counts[i] == 1:
36                cnt += uf.sz[i]
37
38        if cnt > max_impact_num or (cnt == max_impact_num and v < ans):
39            max_impact_num = cnt
40            ans = v
41
42    return ans

Review

Andrw Mallett 的 Linux service management 课程第一章讲的就是 BIND DNS。

rpm 包叫做 bind，而服务叫做 named 

yum install bind bind-utils

systemctl enable named --now

netstat -ltn 
# by default both tcp v4/v6 on port 53 is enabled
# port 953 is a managed port used by R(emote)N(name)D(aemon)C(ontrol)

dig www.google.com @127.0.0.1

/etc/named.conf 文件:

  • 默认设定只在 127.0.0.1 地址上的 52 端口上监听，与 localhost 上的查询请求；
  • 可将上述更为需要的 IP 地址，关闭 ipv6 监听，设定可以接受局域网内部的查询请求；
  • DNS 运行模式 cached 或是 forward；
  • 指定域名的 zone 文件；
options {
  listen-on port 53 { any; };
  listen-on-v6 port 53 { none; }; 
  directory "/var/named";
  dump-file "/var/named/data/cached_dump.db";
  allow-query { localhost; 192.168.56.0/24; localnets; };
  forwarders { 8.8.8.8; 8.8.4.4; 1.1.1.1; };
  ## forward only;
};

zone "example.vm." {
  type master;
  file "db.example";
  allow-update { none; };
};

named-checkconf 工具用于查看配置文件是否正确；

下面是一个典型 zone 文件 /var/named/db.sample，将用户组设为 named
