Algorithm

leetcode #104 Maximum Depth of Binary Tree 计算给定 Tree 的最大深度。

最简单的递归解法：

def max_depth(root: TreeNode) -> int:
    if root is None:
        return 0
    return 1 + max(max_depth(root.left), max_depth(root.right))

Tip

QEMU 是模拟器，是虚拟机运行的基础：假如宿主服务器本身的硬件不支持虚拟化扩展（比如 Intel 的 VT-x 或 AMD 的 AMD-v），就由它翻译虚拟机指令给物理 CPU ，即 Type-2 Hypervisor。

KVM 提供性能加速：对于支持相关拓展的物理 CPU 和特定的 Guest VM 类型，它可以将虚拟机 CPU 直接映射到物理 CPU 上，减少额外的翻译开销，这样实现了 Type-1 Hypervisor。如果用 lsmod 查看，可以看到核心模块 kvm.ko 和 任意一个处理器模块 kvm-intel.ko 或 kvm-amd.ko 被载入内核。

libvirt 提供虚拟平台的API，支持多种流行的虚拟化技术：LXC, QEMU，VBox，VMWare ESX，HyperV，Xen 等等。比如 启动 rhel 上的 libvirtd 服务，就自动查看 /etc/libvirt/qemu/networks/ 下面的 xml 配置文件（注意不是 /etc/sysconfig/network-scripts/)，然后创建对应的 bridge 和 nic。此外，virsh 也是另一个常用的管理工具。

virsh 可以同时管理本地、远程宿主服务器。

virsh 其实执行的是 virsh -c qemu:///system，如果管理远程服务器，可以执行 virsh -c qemu+ssh://user@remoteKVMHost/system

利用 virt-top 工具可以查看虚拟机的 CPU，MEM，网络，磁盘等的性能。

如果两台宿主服务器都挂载一个网络存储池，比如同一个 NFS。而且 Guest VM 创建在这个池中，这样就可以实现动态迁移。唯一需要注意的是，最好将 cache 设置为 none

virsh migrate --live GuestName DestinationURL
virsh migrate --live guest1-rhel7-64 qemu+ssh://host2.example.com/system

Redhat 上个月发布了 Universal Base Image ：

  • 不但可运行在 Redhat 7⁄8 平台，也可以用在其他任何兼容 OCI 的容器平台；
  • 通用型的 baseimage 有 ubi-minimal，ubi，ubi-init；
  • 含特定语言运行时的 baseimage 支持 nodejs, python, ruby；
  • Redhat 会在提供一个专有的 yum repo ，以便客户能自行及时、按需更新软件和安全补丁；
